s - security

Consider following principles:
1. (sbc) Pin specific version within build
    1.1 (sbc) Base images (either major+minor OR SHA256 cash)
    1.2 (sc) Pin specific system dependencies but do not install what is not necessary
    1.3 (sc) Application dependencies - within files intall exact dependencies every time
2. (sb) Use small and secure base images
3. (bc) Protect the layer cache
    3.1 (b) Order commands by frequency of change
    3.2 (b) Copy dependency requirements file -> install dependencies -> copy remaining source code
    3.3 (b) Use cache mounts
    3.4 (b) Use COPY --link
    3.5 (bc) Combine steps that are always linked (use heredocs to improve tidiness)
4. (sc) Be explicit
    4.1 (c) Set working directory with WORKDIR
    4.2 (c) Indicate standard port with EXPOSE
    4.3 (sc) Set default environment variables with ENV
5. (sbc) Avoid unnecessary files
    5.1 (sbc) Use .dockerignore
    5.2 (sbc) COPY specific files
6. (s) Use non-root user
7. (sbc) Install only production dependencies unless specified otherwise in task
8. (s) Avoid leaking sensetive information (e.g. if credentials are required to login into private repository 
they should be passed in a specific way so they don't get built into a layer within container image)
    8.1 (s) Mount secrets (e.g. --mount=type=secret) in order to avoid leakage in build-time
9. (sc) Leverage multi-stage builds 
10. (c) buildx for multi-architecture images
